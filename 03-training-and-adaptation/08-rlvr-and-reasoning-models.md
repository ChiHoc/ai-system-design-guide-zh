# 训练推理模型：RLVR 与 GRPO

前沿推理模型（OpenAI 的 o 系列、DeepSeek-R1、Qwen 的思考模型）采用的训练配方不同于 [RLHF 和 DPO](04-rlhf-and-dpo.md) 中介绍的 RLHF 与 DPO：**带可验证奖励的强化学习（reinforcement learning with verifiable rewards，RLVR）**。取一个能力较强的基础模型，让它在答案*可检查*的问题（数学、代码、逻辑）上生成较长的思维链，并用程序化验证器直接奖励正确性，而不是使用学习得到的奖励模型。本章介绍 RLVR、DeepSeek 推广开来的 GRPO 算法、奖励设计的失败模式、关于 RL 到底增加了什么能力的当前争论，以及多数团队应该优先采用的更便宜的蒸馏路径。

## RLVR：带可验证奖励的强化学习

RLVR 用确定性的**验证器**替代 RLHF 中学习得到的奖励模型，该验证器通过真实答案检查返回奖励：最终答案是否匹配（数学）、代码是否通过单元测试、证明是否通过检查。策略是 LLM；环境是一次单轮生成，即先生成较长的思维链再给出答案；奖励则通过数值/字符串匹配或运行代码来计算。

**为什么“可验证”很重要：它在很大程度上绕开了奖励模型黑客攻击。** 在 RLHF 中，奖励模型本身是一个基于偏好训练的神经网络，策略会学习利用它的盲点（Goodhart / 过度优化问题）。而针对“这是否等于标准答案”或“是否所有测试都通过”的验证器，没有同样容易被博弈的柔性、可学习表面；奖励建立在外部预言机之上。这是 RLVR 在推理任务上比 RLHF 扩展得更稳定的核心原因。它并不*消除*博弈（见[奖励设计](#奖励设计与失败模式)），但移除了最容易利用的漏洞。

每一步循环如下：
1. 采样带有已知可检查答案的提示。
2. 对每个提示，从当前策略采样一个或多个长 CoT（chain-of-thought，思维链）补全。
3. 对每个补全运行验证器，得到一个标量奖励（通常是 1 正确 / 0 错误，有时再加一个很小的格式奖励）。
4. 计算优势并执行策略梯度更新。
5. 重复。长思维链、自我验证和回溯是从这个循环中*涌现*出来的，而不是被显式监督出来的。

典型的开放成果是 **DeepSeek-R1**（arXiv:2501.12948；同行评审版本于 2025 年 9 月发表在 Nature），其报告称，推理可以通过*纯 RL* 被激励出来，不需要人工标注的推理轨迹，并出现了自我反思的涌现。一个教学上的注意点：OpenAI 尚未公开 o 系列算法，因此应把“o 系列使用 RLVR 风格的 RL”视为基于公开信息的合理推断，而不是已记录的事实，也不要把某个具体算法归因于它。

## GRPO

**组相对策略优化（Group Relative Policy Optimization，GRPO）** 最早在 DeepSeekMath（arXiv:2402.03300）中提出，并被用作 DeepSeek-R1 的 RL 引擎。

**它移除了 PPO 的问题。** PPO 需要一个**价值/评论器网络（value/critic network）**，通常与策略模型同等规模，用于估计优势计算中的基线。对于 LLM 来说，这大致会让内存翻倍，并增加第二个需要训练的模型；同时，对带有单一终止奖励的长序列学习价值估计也很困难。GRPO 的做法是：**删除评论器。** 它不使用学习得到的基线，而是对同一个提示采样一**组**补全，并用这组补全自身的奖励统计量作为基线。

**组相对优势。** 对一个提示，采样补全 `o_1..o_G`，得到奖励 `r_1..r_G`。补全 `i` 的优势是它在组内的 z-score，并广播到该补全中的每个 token：

```python
# One GRPO step. policy = the LLM; verify() = a programmatic checker.
for q in sample_prompts(batch):
    group = [policy.generate(q) for _ in range(G)]      # G group, e.g. 8..64
    R = [verify(q, o) for o in group]                    # verifiable reward, e.g. 1.0 / 0.0
    mu, sigma = mean(R), std(R)
    A = [(r - mu) / (sigma + 1e-6) for r in R]           # group-relative advantage (the critic-free baseline)
    for o_i, A_i in zip(group, A):
        ratio = exp(policy.logprob(o_i) - old_policy.logprob(o_i))  # per-token prob ratio
        loss += -mean(min(ratio * A_i, clip(ratio, 1-eps, 1+eps) * A_i))
        loss += beta * KL(policy, ref_policy)            # original GRPO keeps a KL term
update(policy, loss)
```

DeepSeek 使用它，是因为在前沿规模下去掉评论器能带来实际的内存收益，而组基线也适合单一终止可验证奖励。

**已知问题，以及修复它们的变体：**
- **长度偏差。** 因为一个标量优势会广播到每个 token，较长的补全会把它的优势分摊到更多 token 上，优化过程可能会膨胀响应长度（尤其是错误答案）。**Dr.GRPO**（arXiv:2503.20783）移除了导致这一问题的长度与标准差归一化项。
- **零方差坍缩。** 如果一组中的所有补全都得到相同奖励（在简单提示上全都正确，或在困难提示上全都错误），标准差会趋近于零，每个优势都会变为零，该组就*没有梯度*。随着模型改进，更多提示会变成“全都正确”，因此有效批量会缩小。**DAPO**（arXiv:2503.14476）通过**动态采样**修复这一点：过采样并过滤掉任何全正确或全错误的提示组，只保留能产生梯度的组。DAPO 还解耦了裁剪范围（“clip-higher”）、使用 token 级损失，并去掉 KL 项。
- **MoE 不稳定性。** 对混合专家模型（mixture-of-experts，MoE）来说，token 级重要性比率噪声较大，因为不同策略版本之间的路由不同。**GSPO**（arXiv:2507.18071，Qwen3 中使用）改为在整段序列似然上定义比率，从而稳定 MoE 的 RL 训练。

实践经验是：将提示筛选在模型的难度区间内，并优先选择去偏的变体（Dr.GRPO、DAPO 或 GSPO），而不是原版 GRPO。

## 奖励设计与失败模式

即使使用可验证奖励，策略优化的也是验证器*实际*测量的东西，而不是你本来想要的东西。

- **博弈验证器。** 模型可以在没有学会模式的情况下通过检查器（例如，枚举实例级标签），这是一种已有记录的 RLVR 奖励黑客模式。
- **“奇迹步骤”。** 思维链可能通过*不忠实*的中间步骤得到正确的最终答案（arXiv:2510.07774）。纯结果验证器会奖励正确答案，同时也一起奖励糟糕推理。
- **格式奖励与正确性奖励。** R1-Zero 风格的配方会把一个小的格式奖励（把推理放在带标签的块中，给出可解析答案）与正确性奖励配对。如果格式奖励相对正确性过大，模型就会优化这种廉价、可博弈的信号。
- **长度利用**，既包括上面提到的 GRPO 梯度伪影，也包括“冗长程度与通过率相关”的真实黑客方式。缓解方法包括：token 级损失、过长响应惩罚、长度归一化目标。

一个需要标记的重要注意点：**虚假奖励（Spurious Rewards）** 发现（arXiv:2506.10947）报告称，在某个模型家族上，使用*随机*甚至*错误*奖励的 GRPO 也产生了很大收益，原因是它放大了该家族预训练中已经潜藏的行为。关键在于，该论文强调这些收益**没有**迁移到其他模型家族。教训是：必须对单一模型家族上的 RLVR 结果保持高度警惕，并始终加入来自不同模型家族的基线。

## RL 是增加能力还是优化采样？

这是本章最微妙的一点：一个真正开放的问题，目前有三种立场。

- **RL（强化学习）只是优化采样，并不增加能力。** 一条研究路线（arXiv:2504.13837）报告称，RLVR（带可验证奖励的强化学习）模型在 pass@1 上优于基础模型，但基础模型在较大的 pass@k 上追平或超过它们，这表明 RL 模型使用的每条推理路径其实已经存在于基础模型中；RL 提高了采样到正确推理链的概率，但缩小了可探索集合。
- **长期 RL 会扩展能力边界。** 反向结果（ProRL，arXiv:2505.24864；以及 arXiv:2506.14245，后者引入了推理感知的 CoT-Pass@K 指标）报告称，只要 RL 足够充分，模型就能解决基础模型在所有 k 下都失败的问题，并且 RLVR 提升的是*推理正确*的答案，而不只是碰巧正确的最终答案。
- **两者都对，分两个阶段（应优先采用的综合观点）。** 一种调和观点（arXiv:2510.04028）发现，早期 RL 会利用并优化已有能力（pass@1 上升，pass@k 持平），而长期 RL 会探索并扩展边界。最清晰的实践综合结论（arXiv:2512.07783）报告称，RL 只有在预训练留下提升空间、且 RL 数据瞄准模型能力边缘时，才会产生真实增益（以高 pass@k 衡量），也就是那些模型在 pass@1 下失败、但在 pass@k 下有时能做对的问题。对已经解决或毫无希望的问题做 RL 收益很小。

同一项工作还强调了**中期训练**：这是介于通用预训练和 RL 之间的阶段，会在一个经过策划、推理密集的混合数据上继续训练，以巩固正在浮现的技能。在固定计算预算下，它通常优于只做 RL，因为它提高了基础能力，而 RL 随后会进一步优化这些能力。诚实的解读是：领域共识已经趋向于“RL 在能力边缘效果最好”，但 RL 是否能制造出*真正新颖*的能力仍然存在争议。

## 蒸馏：更便宜的路径

对大多数团队来说，RL 不是正确的第一步。

**离策略蒸馏（在教师轨迹上做 SFT）。** DeepSeek-R1 报告称，用 R1 生成的大约 800K 条推理轨迹对稠密 Qwen 和 Llama 模型进行微调，并且蒸馏后的 32B 模型优于直接在同一个 32B 基础模型上运行大规模 RL；论文指出，后者需要多得多的计算量，但仍然无法匹配蒸馏效果。教训是：对于已知领域中的小模型，从强推理模型蒸馏比自己做 RL 更有效，也便宜得多。

**在线策略蒸馏**（Thinking Machines Lab，2025）用学生模型*自己*采样出的补全进行训练（在线策略，类似 RL），但使用来自教师分布的稠密逐 token 监督（类似 SFT），而不是稀疏的标量奖励。报告结果显示，在灌输推理能力方面，它的样本效率大约比 RL 高一个数量级，把在线策略相关性与稠密、稳定的信号结合起来。截至 2026，这些数字主要来自单一厂商，因此应把具体倍数视为报告值，而非已确立结论。

| 方法 | 轨迹 | 奖励信号 | 成本 / 稳定性 |
|--------|-------------|---------------|------------------|
| SFT / 离策略蒸馏 | 教师的 | 稠密 token 目标 | 便宜、稳定；存在一些暴露偏差 |
| RL / RLVR | 学生自己的 | 稀疏标量 | 昂贵、较不稳定；在线策略 |
| 在线策略蒸馏 | 学生自己的 | 稠密（教师逐 token） | 在线策略相关性加稠密信号 |

更广泛的讨论见[知识蒸馏](05-knowledge-distillation.md)。

## 实践指导

**训练自己的推理模型，还是使用现成模型。** 对大约 95% 的团队来说：使用一个强推理模型，或从它蒸馏。只有在以下条件同时满足时，才自己做 RLVR：(a) 你的领域有干净的程序化验证器，(b) 基础模型或教师模型确实无法被提示到你的目标行为，(c) 你有 RL 基础设施、GPU 预算以及对不稳定性的容忍度，(d) 你的任务位于模型的能力边缘。

**粗略需求（数量级，不是承诺）：** 蒸馏需要 10^5 到 10^6 条高质量教师轨迹，以及标准 SFT 计算量；RLVR 需要数千到数万条*可验证*提示，并且要围绕难度区间精心策划，其中 rollout（而不是梯度步骤）是成本瓶颈。据报告，DeepSeek-R1 的后训练大约消耗 147K 个 H800 GPU 小时，远超业余规模，尽管低于通常想象。

**评估陷阱：** 同时报告 pass@1 *和* pass@k（单独的 pass@1 会隐藏采样与能力之间的区别）；使用多个基础模型家族（虚假奖励伪影）；检查推理忠实性，而不只是最终答案；把响应长度漂移当作奖励黑客的烟雾报警器；优先使用留出集或新鲜竞赛集来防范基准污染。见[基准和排行榜](../14-evaluation-and-observability/03-benchmarks-and-leaderboards.md)。

**成熟度：** 针对数学和代码、且配有干净验证器的蒸馏与 GRPO/DAPO 配方已经成熟且可复现。在线策略蒸馏、用于 MoE（混合专家模型）的 GSPO，以及作为有意设计阶段的中期训练正在快速成熟。RLVR 是否增加新能力，以及如何在不重新引入奖励模型黑客的情况下，将它干净地扩展到可验证领域之外，仍然真正未定。因此，应锚定机制（组基线、长度归一化、动态采样、稠密监督与稀疏监督、能力边缘数据），而不是任何单一排行榜数字。

## 面试题

### 问：什么是 RLVR，为什么它在推理任务上比 RLHF 扩展得更稳定？

**优秀回答：**
RLVR 是带可验证奖励的强化学习：你不用学习出来的奖励模型，而是用确定性的验证器来奖励策略，例如数学答案是否匹配、代码是否通过测试。它比 RLHF（基于人类反馈的强化学习）扩展得更稳定，因为奖励基于外部判定器，而不是策略可以过度优化的神经网络。RLHF 的经典失败模式是奖励模型黑客，也就是策略找到一些奖励模型打高分、但人类并不认可的输入；单元测试通过则没有类似可利用的软表面。代价是，RLVR 只适用于可以机械检查正确性的地方，这也是它最先在数学、代码和逻辑领域兴起的原因。它并不能消除投机取巧，模型仍然可能通过不忠实的步骤得到正确答案，或钻验证器的空子，所以仍然要关注忠实性和长度。

### 问：解释 GRPO，并说明它的一个已知失败模式。

**优秀回答：**
GRPO 去掉了 PPO 的价值/评论家网络，而对 LLM 来说，这个网络大约会让内存翻倍；它用组相对基线取代学习到的基线：对同一个提示采样多个补全，用验证器给每个补全打分，并把组内 z 分数作为优势，广播到每个 token。这让它成本较低，并且非常适合只有单一终端可验证奖励的场景，这也是 DeepSeek 使用它的原因。一个关键失败模式是零方差坍缩：当一个组中的每个补全都得到相同奖励时，无论是简单提示上全都正确，还是困难提示上全都错误，标准差都是零，优势也是零，这个组不会产生梯度。随着模型改进，更多提示会变成全正确，因此有效 batch 会缩小。DAPO 的动态采样通过过滤掉全正确和全错误的组，只在真正产生梯度的组上训练，从而修复这一点。此外，把一个优势广播到所有 token 还会带来长度偏置，Dr.GRPO 则通过移除长度归一化来处理这一问题。
