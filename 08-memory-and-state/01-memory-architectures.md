# 记忆架构

LLM（大语言模型，Large Language Model）记忆已经从“历史缓冲区”演进为**三层认知架构（Three-Tiered Cognitive Architecture）**。这一层级模仿人类认知系统（L1-L3），以平衡速度、成本和回忆容量。生产环境中的智能体（agent）技术栈如今更倾向于在向量存储之上使用专用记忆层（Mem0、Zep、Letta、Cognee），而不是自行从头实现。

## 三层层级

| 层级 | 类型 | 人类类比 | 技术实现 | 延迟 |
|------|------|----------|----------|------|
| **L1** | 工作记忆 | 即时关注焦点 | 上下文窗口 / KV Cache（键值缓存） | <50ms |
| **L2** | 情景记忆 | 过往经历 | 向量数据库 / 本地图谱 | 100-300ms |
| **L3** | 语义记忆 | 通用知识 | 全局图谱 / SQL / Mem0 | >500ms |

---

## 第一层：工作记忆（L1）

L1 是模型的**当前活跃焦点**。  
- **上下文窗口**：当前前沿模型上的上下文长度为 128K - 2M tokens（Claude Opus 4.7、Claude Sonnet 4.6、GPT-5.5、Gemini 3.1 Pro）。
- **KV Cache**：存储预计算键和值的 GPU“内存”。
- **管理策略**：**滑动窗口**和**前缀缓存**（vLLM/PagedAttention）。
- **冗余说明**：我们只在 L1 中保留最近的轮次和关键系统指令。

---

## 第二层：情景记忆（L2）

L2 存储本次会话或与该用户的过往会话中“之前发生了什么”。
- **存储**：向量数据库（Pinecone、Weaviate、Qdrant）。
- **检索**：语义搜索。如果用户问“我们上周二聊了什么？”，L2 会提供答案。
- **模式**：**经验回放**。智能体检索过去成功的轨迹来指导当前决策。

---

## 第三层：语义记忆（L3）

L3 存储**不可变事实**和**已学习规则**。
- **知识图谱**：存储关系，例如 `User` -- `WORKS_FOR` --> `Company_X`。
- **Mem0**：一种托管服务，会抽取事实（例如“User likes Dark Mode”）并使其可在全局范围内使用。
- **真值锚定**：当 L1 和 L2 提供冲突信息时，L3 充当“基准真值（Ground Truth）”。

---

## 记忆巩固模式

记忆通过**巩固**在各层之间迁移：
1. **抽取**：会话结束时，LLM“审阅者”从 L1 中抽取事实。
2. **索引**：事实被存入 L2（作为向量）和 L3（作为图节点）。
3. **衰减**：未被强化的旧记忆会从 L2 移到冷存储（L3）或被删除。

## 面试题

### Q: 为什么不直接用 2M token 的上下文窗口来承载全部记忆（L1-L3）？

**标准答案：**
虽然可行，但这在**经济上和认知上都低效**。  
1. **成本**：每一轮都用 1M+ tokens 调用模型，成本会显著高于使用 RAG 召回上下文的 10K token 调用。
2. **注意力稀释**：即使是“长上下文”模型，“中间遗忘（Lost in the Middle）”仍然是一个因素。如果上下文中塞满无关的历史轮次，模型对**当前**任务的推理会退化。
3. **延迟**：TTFT（Time to First Token，首个 token 时间）会随上下文规模增大而增加，因为需要加载 KV cache。  
高级别架构会使用**策略性检索**，让上下文窗口保持精简且聚焦。

### Q: 如何处理第三层（全局语义记忆）中的“隐私泄露（Privacy Leakage）”？

**标准答案：**
第三层（语义记忆）必须按**命名空间分片**。每个用户或组织在向量数据库和知识图谱中都会获得唯一的 `namespace_id`。我们在数据库层实现**RLS（Row Level Security，行级安全）**。此外，我们在巩固步骤中使用**PII 脱敏层**，以确保敏感数据（passwords、PII）永远不会从临时的 L1 上下文进入持久化的 L3 知识存储。
