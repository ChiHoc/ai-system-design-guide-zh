# PagedAttention

PagedAttention（分页注意力）是高吞吐量服务引擎（vLLM、SGLang、TensorRT-LLM）背后的基础算法。它解决了此前限制 LLM 可扩展性的“内存碎片化”问题。

## 连续内存问题

标准深度学习框架会以较大的连续块分配内存。  
对于一个 LLM 请求，你可能会为 8192 个 token 的 `max_sequence_length` 预留内存。

**浪费：**
1. **内部碎片**：如果用户只生成 10 个 token，那么该预留块中有 99.9% 被浪费了。
2. **外部碎片**：即使可用内存总量很高，内存也会被分割成太小的空隙，无法容纳一个新的“大块”。

---

## PagedAttention 如何工作（vLLM）

PagedAttention 的灵感来自操作系统中的虚拟内存。

1. **Token 到块**：一个请求的 KV 缓存被拆分为小的、固定大小的 **块**（例如每个块 16 个 token）。
2. **逻辑与物理**：模型认为自己在关注一个连续序列（逻辑内存），但这些块实际上分散在 VRAM（物理内存）中。
3. **查找表**：**块表**将逻辑索引映射到物理地址。

**主要收益**：内存浪费从约 60-80% 降低到 **低于 4%**。

---

## 虚拟内存管理（块管理器）

服务框架（vLLM、SGLang）充当 GPU 的“迷你操作系统”。

- **分配**：当一个新请求开始时，块管理器会为它分配一组空的物理块。
- **驱逐**：如果 VRAM 已满，管理器可以将不活跃的 KV 块“交换”到 CPU RAM，并在需要时把它们再取回（分页交换）。

---

## KV 缓存共享（写时复制）

PagedAttention 使“公共前缀”的共享变得轻而易举。

**场景**：100 个用户正在与同一个 5,000 个 token 的系统提示词聊天。
- **传统方式**：将那个 5,000 个 token 的 KV 缓存存储 100 次（在 VRAM 中占用 **500k tokens**）。
- **PagedAttention**：通过块表将其 **只存一次**，并让所有 100 个用户指向同一组物理块。
- **写时复制**：如果某个用户生成了一个独有 token，就只为他新建一个块，而共享块保持不变。

## 面试题

### Q: 为什么 PagedAttention 能显著提高吞吐量？

**强回答：**
PagedAttention 通过支持更大的 **batch size（批大小）** 来提高吞吐量。由于它消除了内部和外部内存碎片，我们可以在同样的 GPU VRAM 中塞入更多请求。在传统服务中，我们可能只能容纳 4 个请求，因为必须“预留”最大长度的块；而使用 PagedAttention 时，我们可以容纳 20-30 个请求，因为我们只为实际存在的 token 使用内存。更大的批次带来更好的 GPU 利用率，以及显著更高的总体每秒 token 数。

### Q: 在 vLLM 的语境下解释“块表（Block Table）”。

**强回答：**
块表是一种映射结构，用来连接模型对连续数据的期望与分散内存的物理现实。表中的每一项对应一个 token 的“逻辑块”。它存储该块的 key 和 value 张量所在 GPU 内存的物理地址。这使框架能够以小块粒度动态分配和释放内存，并支持前缀共享和高效多线程等高级特性。
